# Backend Database (Supabase / PostgreSQL)

Odyssey uses **Supabase Postgres** with **Row Level Security (RLS)** enabled on all user data tables.
Each table restricts access so users can **only read and modify their own data**.

## 1.1 `semesters`

Stores semester planning information for a user.

```sql
create table public.semesters (
  id bigint generated always as identity primary key,

  user_id uuid
    references auth.users(id)
    on delete cascade,

  start_date date not null,
  end_date date not null,

  total_goal_hours numeric not null,
  total_study_days integer not null,

  daily_required_hours numeric not null,
  weekly_required_hours numeric not null,

  inserted_at timestamptz default now()
);
```

### Index

```sql
create index if not exists semesters_user_id_idx
on public.semesters(user_id);
```

<br>


## 1.2 `subjects`

Stores subjects linked to a semester.

```sql
create table public.subjects (
  id bigint generated always as identity primary key,

  semester_id bigint not null
    references public.semesters(id)
    on delete cascade,

  name text not null,
  target_hours numeric not null,
  weight numeric,

  inserted_at timestamptz default now()
);
```

<br>



## 1.3 `sessions`

Stores all study sessions (pomodoro, stopwatch, or manual).

```sql
create table public.sessions (
  id bigint generated by default as identity primary key,

  user_id uuid not null
    references auth.users(id)
    on delete cascade,

  semester_id bigint not null
    references public.semesters(id)
    on delete cascade,

  subject_id bigint
    references public.subjects(id),

  name text,

  type text not null, -- 'pomodoro' | 'stopwatch' | 'manual'

  start_time timestamptz not null,
  end_time timestamptz,

  duration_minutes numeric not null,

  notes text,

  inserted_at timestamptz default timezone('utc', now())
);
```


<br>



## 1.4 `user_rewards`

Tracks reward unlock counts and last unlock date.

```sql
create table public.user_rewards (
  id bigint generated always as identity primary key,

  user_id uuid not null
    references auth.users(id)
    on delete cascade,

  reward_key text not null,

  unlock_count integer not null default 0,
  last_unlocked_date date,

  inserted_at timestamptz default timezone('utc', now()),

  unique (user_id, reward_key)
);
```

<br>



## 1.5 `daily_journal`

Stores user journal notes and mood for each day.

```sql
create table public.daily_journal (
  id uuid primary key default gen_random_uuid(),

  user_id uuid not null,

  date_key text not null,

  note text,
  mood text,

  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

### Unique Constraint

Each user can have **only one journal entry per day**.

```sql
create unique index daily_journal_unique_day
on public.daily_journal (user_id, date_key);
```

# 2. Row Level Security (RLS)

All tables enforce **user-level access control**.

Enable RLS:

```sql
alter table public.semesters enable row level security;
alter table public.subjects enable row level security;
alter table public.sessions enable row level security;
alter table public.user_rewards enable row level security;
alter table public.daily_journal enable row level security;
```

# 3. RLS Policies



# 3.1 Semesters Policies

```sql
create policy "Users can view own semesters"
on public.semesters
for select
using (auth.uid() = user_id);
```

```sql
create policy "Users can insert own semesters"
on public.semesters
for insert
with check (auth.uid() = user_id);
```

```sql
create policy "Users can update own semesters"
on public.semesters
for update
using (auth.uid() = user_id)
with check (auth.uid() = user_id);
```

```sql
create policy "Users can delete own semesters"
on public.semesters
for delete
using (auth.uid() = user_id);
```

<br>



# 3.2 Subjects Policies

Subjects are accessed **only if the semester belongs to the user**.

### Select

```sql
create policy "Users can view own subjects"
on public.subjects
for select
using (
  exists (
    select 1
    from public.semesters
    where semesters.id = subjects.semester_id
    and semesters.user_id = auth.uid()
  )
);
```

### Insert

```sql
create policy "Users can insert own subjects"
on public.subjects
for insert
with check (
  exists (
    select 1
    from public.semesters
    where semesters.id = subjects.semester_id
    and semesters.user_id = auth.uid()
  )
);
```

### Update

```sql
create policy "Users can update own subjects"
on public.subjects
for update
using (
  exists (
    select 1
    from public.semesters s
    where s.id = subjects.semester_id
    and s.user_id = auth.uid()
  )
)
with check (
  exists (
    select 1
    from public.semesters s
    where s.id = subjects.semester_id
    and s.user_id = auth.uid()
  )
);
```

### Delete

```sql
create policy "Users can delete own subjects"
on public.subjects
for delete
using (
  exists (
    select 1
    from public.semesters s
    where s.id = subjects.semester_id
    and s.user_id = auth.uid()
  )
);
```

# 4. Security Design

The backend follows these security rules:

* All user data tables enforce **Row Level Security**
* Every query checks `auth.uid()`
* Cross-table access (like subjects → semesters) uses **EXISTS validation**
* Foreign keys enforce referential integrity
* Cascading deletes remove related data when a user or semester is deleted

This ensures **users can only access their own academic data, sessions, and rewards**.

```sql



CREATE TABLE public.daily_journal (
  id uuid NOT NULL DEFAULT gen_random_uuid(),
  user_id uuid NOT NULL,
  date_key text NOT NULL,
  note text,
  mood text,
  created_at timestamp with time zone DEFAULT now(),
  updated_at timestamp with time zone DEFAULT now(),
  CONSTRAINT daily_journal_pkey PRIMARY KEY (id)
);


CREATE TABLE public.semesters (
  id bigint GENERATED ALWAYS AS IDENTITY NOT NULL,
  start_date date NOT NULL,
  end_date date NOT NULL,
  total_goal_hours numeric NOT NULL,
  total_study_days integer NOT NULL,
  daily_required_hours numeric NOT NULL,
  weekly_required_hours numeric NOT NULL,
  inserted_at timestamp with time zone DEFAULT now(),
  user_id uuid,
  CONSTRAINT semesters_pkey PRIMARY KEY (id),
  CONSTRAINT semesters_user_id_fkey FOREIGN KEY (user_id) REFERENCES auth.users(id)
);


CREATE TABLE public.sessions (
  id bigint GENERATED ALWAYS AS IDENTITY NOT NULL,
  user_id uuid NOT NULL,
  semester_id bigint NOT NULL,
  subject_id bigint,
  name text,
  type text NOT NULL,
  start_time timestamp with time zone NOT NULL,
  end_time timestamp with time zone,
  duration_minutes numeric NOT NULL,
  notes text,
  inserted_at timestamp with time zone DEFAULT timezone('utc'::text, now()),
  CONSTRAINT sessions_pkey PRIMARY KEY (id),
  CONSTRAINT sessions_user_id_fkey FOREIGN KEY (user_id) REFERENCES auth.users(id),
  CONSTRAINT sessions_semester_id_fkey FOREIGN KEY (semester_id) REFERENCES public.semesters(id),
  CONSTRAINT sessions_subject_id_fkey FOREIGN KEY (subject_id) REFERENCES public.subjects(id)
);


CREATE TABLE public.subjects (
  id bigint GENERATED ALWAYS AS IDENTITY NOT NULL,
  semester_id bigint NOT NULL,
  name text NOT NULL,
  target_hours numeric NOT NULL,
  weight numeric,
  inserted_at timestamp with time zone DEFAULT now(),
  CONSTRAINT subjects_pkey PRIMARY KEY (id),
  CONSTRAINT subjects_semester_id_fkey FOREIGN KEY (semester_id) REFERENCES public.semesters(id)
);


CREATE TABLE public.user_rewards (
  id bigint GENERATED ALWAYS AS IDENTITY NOT NULL,
  user_id uuid NOT NULL,
  reward_key text NOT NULL,
  unlock_count integer NOT NULL DEFAULT 0,
  last_unlocked_date date,
  inserted_at timestamp with time zone DEFAULT timezone('utc'::text, now()),
  CONSTRAINT user_rewards_pkey PRIMARY KEY (id),
  CONSTRAINT user_rewards_user_id_fkey FOREIGN KEY (user_id) REFERENCES auth.users(id)
);
```
